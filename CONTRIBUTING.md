# Contributing

Thank you for your interest in contributing to **Docker Compose Kit**! We welcome contributions including bug reports, feature requests, documentation improvements, and code changes.

## Pull Requests

- Open an issue first for major changes or new features to discuss the approach and avoid duplication.
- Fork the repository and create a feature branch for your changes.
- Keep your branch up to date with `main` before submitting a pull request.
- Provide a clear title and description for your pull request.
- Link related issues if applicable.

## Commit Messages

We follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) to keep commit history clean:

- Format: `<type>: <subject>`
- Breaking changes must include `!`, e.g., `feat!: ...`
- Common types:
  - `feat`: new feature
  - `fix`: bug fix
  - `docs`: documentation changes
  - `refactor`: code change without new features or fixes
  - `test`: adding/updating tests
  - `chore`: build process or tooling changes

Examples:
- `feat: add example for decoupled frontend/backend`
- `fix: correct volume mount path in Node.js service`
- `docs: update README getting started section`

## Guidelines

- Keep pull requests focused on a single concern.
- Update documentation if your changes affect usage or behavior.
- Be respectful and constructive in discussions and code reviews.

## Contact

For questions or guidance, open an issue in the repository.

Thank you for helping improve **Docker Compose Kit**!
